# Issue Expansion Template

## Purpose
Transform simple ideas from `idea.md` into comprehensive GitHub issue specifications.

## Our Standards
- Follow YAGNI, KISS, DRY principles (see `.claude/PRINCIPLES.md`)
- Focus on requirements, not implementation details
- Be specific and testable

## Template Format
```markdown
# [Feature Name]

## Problem
[Clear 2-3 sentence problem statement]

## User Story
As a [Role], I want [Feature], so that [Benefit].

## Requirements
- [ ] Requirement 1
- [ ] Requirement 2

## Acceptance Criteria
- [ ] Criteria 1
- [ ] Criteria 2

## Verification & Testing
### Automated Tests
- [ ] Unit Tests: [Describe specific unit tests, e.g., 'Test user model creation' or 'Mock API response']
- [ ] Integration Tests: [Describe integration flows if applicable]

### Manual Verification
- [ ] [Step-by-step manual verification instruction]
[Any specific constraints or implementation ideas]
[Free to add more than three items to acceptance criteria and requirements.]
```
